# Books API with Redis Cache

## Описание

Приложение для управления книгами с использованием **Spring Boot**, **Spring Data JPA** и **Redis**.

В системе есть 2 сущности:
- **Book** — книга;
- **Category** — категория книги.

Связь между ними — **ManyToOne**: у книги одна категория, у категории много книг.

---

## Что умеет приложение

- найти книгу по **названию и автору**;
- найти список книг по **имени категории**;
- создать книгу;
- обновить книгу;
- удалить книгу по ID;
- кэшировать поисковые запросы в Redis;
- очищать кэш при создании, обновлении и удалении книги.

---

## Как работает кэш

### 1. Поиск книги по названию и автору
Результат поиска сохраняется в Redis с ключом:

```text
title + "_" + author
```

### 2. Поиск книг по категории
Результат поиска списка книг сохраняется в Redis с ключом:

```text
categoryName
```

### 3. Создание, обновление и удаление книги
После изменения данных кэш очищается, чтобы не возвращать устаревшую информацию.

## Эндпоинты

### Получить все книги
```http
GET /api/books
```

### Получить книгу по ID
```http
GET /api/books/{id}
```

### Найти книгу по названию и автору
```http
GET /api/books/search?title=...&author=...
```

### Найти книги по категории
```http
GET /api/books/by-category?categoryName=...
```

### Создать книгу
```http
POST /api/books
```

### Обновить книгу
```http
PUT /api/books/{id}
```

### Удалить книгу
```http
DELETE /api/books/{id}
```

## DTO

### BookRequest
Используется для создания и обновления книги.

Поля:
- `title`
- `author`
- `categoryName`

### BookDTO
Используется в ответах API.

Поля:
- `id`
- `title`
- `author`
- `category`

---

## Категории

Категория создаётся автоматически вместе с книгой, если её ещё нет в базе.

Если категория уже существует, она переиспользуется.

---

## Кэш Redis

В приложении включено кэширование через Redis.

### TTL
Время жизни кэша: **10 минут**.

### Кэшируемые методы
- поиск по названию и автору;
- поиск по категории.

### Инвалидация кэша
- при создании книги;
- при обновлении книги;
- при удалении книги.

---

## Валидация

При создании и обновлении книги проверяются входные данные:

- `title` не должен быть пустым;
- `author` не должен быть пустым;
- `categoryName` не должен быть пустым.

---

## Обработка ошибок

Если книга не найдена, приложение возвращает ошибку `404`.

Если входные данные некорректны, приложение возвращает `400`.
